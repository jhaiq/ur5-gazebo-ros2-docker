# 机器人 AI Agent 框架调研报告

**调研日期:** 2026-04-18  
**调研范围:** 2024-2026 年开源机器人 AI Agent 框架

---

## 一、调研概述

本次调研收集了 7 个主流机器人 AI Agent 框架，涵盖从学术研究到工业应用的不同层次。调研重点分析各框架的架构设计、核心功能、以及与 robot_openclaw 项目的可借鉴点。

---

## 二、框架详细分析

### 2.1 RAI (Robot Agent Interface)

**项目地址:** https://github.com/RobotecAI/rai  
**发布机构:** Robotec AI  
**论文:** [arXiv:2505.07532](https://arxiv.org/abs/2505.07532)

#### 架构特点

| 特性 | 描述 |
|------|------|
| **供应商无关** | 不绑定特定机器人硬件或大模型 |
| **ROS 2 集成** | 深度集成 ROS 2 工具链 |
| **多智能体系统** | 支持复杂的多智能体架构 |
| **多模态交互** | 原生支持多种数据类型和感官输入 |

#### 核心模块

```
rai/
├── rai_core          # 多智能体系统、人机交互、多模态核心功能
├── rai_whoami        # 从 URDF/文档提取机器人具身信息
├── rai_asr           # 语音识别 (Speech-to-Text)
├── rai_tts           # 语音合成 (Text-to-Speech)
├── rai_sim           # 仿真环境连接
├── rai_bench         # 基准测试套件
├── rai_perception    # 基于开放集模型的目标检测
├── rai_nomad         # NoMaD 导航集成
└── rai_finetune      # 具身数据微调 LLM
```

#### 应用场景

| 应用 | 机器人 | 描述 |
|------|--------|------|
| 果园任务规划 | 自主拖拉机 | 障碍推理与最佳路径分析 |
| 自然语言操作 | Franka Panda | 结合 Grounded SAM 2 的灵活操作 |
| 自主移动机器人 | Husarion ROSbot XL | 导航与控制交互 |
| 移动操作复合机器人 | RB-KAIROS | 板载执行的完整演示 |

#### 可借鉴点

1. **rai_whoami 工具**: 自动从 URDF 提取机器人能力描述，值得 robot_openclaw 借鉴
2. **rai_bench 基准测试**: 提供标准化的 Agent 性能评估方法
3. **多模态原生支持**: 统一处理视觉、语音、文本等多种输入

### 2.2 ROSClaw

**论文地址:** [arXiv:2603.26997](https://arxiv.org/abs/2603.26997)  
**发布机构:** 学术界 (OpenClaw 团队)  
**核心贡献:** 模型无关的执行层

#### 核心特性

1. **动态能力发现**: 标准化的 affordance 注入机制
2. **多模态观察归一化**: 统一不同传感器的数据格式
3. **执行前动作验证**: 可配置的安全包络
4. **结构化审计日志**: 完整的操作溯源

#### 架构优势

```
┌─────────────────────────────────────────────────────────┐
│                   Foundation Model                       │
│  (可插拔：任何大模型后端，配置切换)                        │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                    ROSClaw 执行层                         │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐        │
│  │ 能力发现     │ │ 安全验证     │ │ 审计日志     │        │
│  └─────────────┘ └─────────────┘ └─────────────┘        │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│                      ROS 2 机器人                         │
│  (可插拔：轮式/四足/人形，配置切换)                        │
└─────────────────────────────────────────────────────────┘
```

#### 实验结果

- 部署于 3 种平台 (轮式、四足、人形)
- 支持 4 种大模型后端
- 模型间 out-of-policy 动作提案率差异达 4.8 倍
- 相同指令下不同模型产生质的物理行为差异

#### 可借鉴点

1. **模型无关设计**: robot_openclaw 应支持任意大模型后端
2. **安全包络机制**: 在执行前验证动作是否在安全范围内
3. **审计日志**: 完整的操作记录便于调试和合规

### 2.3 RoboOS

**项目地址:** https://flagopen.github.io/RoboOS/  
**发布机构:** BAAI (北京智源)  
**论文:** [arXiv:2505.03673](https://arxiv.org/abs/2505.03673)

#### 架构创新：脑 - 小脑分层架构

```
┌─────────────────────────────────────────────────────────┐
│              Embodied Brain (具身大脑)                    │
│  • 多模态大语言模型 (MLLM)                                │
│  • 全局感知与高层决策                                      │
└─────────────────────────────────────────────────────────┘
                          ↓ 分层信息流
┌─────────────────────────────────────────────────────────┐
│           Cerebellum Skill Library (小脑技能库)           │
│  • 模块化、即插即用工具包                                  │
│  • 多技能无缝执行                                         │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│          Real-Time Shared Memory (实时共享内存)          │
│  • 时空同步机制                                          │
│  • 多智能体状态协调                                        │
└─────────────────────────────────────────────────────────┘
```

#### 核心组件

| 组件 | 功能 |
|------|------|
| Embodied Cloud Model | 多模态大语言模型，全局感知和高层决策 |
| Cerebellum Skill Library | 模块化技能库，支持即插即用 |
| Real-Time Shared Memory | 多智能体状态同步机制 |

#### 工作流程

1. **分层任务分解**: 将复杂任务分解为子任务
2. **拓扑感知子任务分配**: 根据机器人能力分配任务
3. **分布式代理执行**: 多机器人并行执行
4. **动态内存更新**: 实时更新共享状态

#### 应用场景

- **餐厅**: Unitree G1 + Agilex 协作准备汉堡
- **家庭**: Realman + Agilex 协作取物
- **超市**: 多机器人协作礼品选择和包装

#### 可借鉴点

1. **脑 - 小脑分层**: 高层规划与底层执行分离，适合 robot_openclaw 参考
2. **实时共享内存**: 多机协同的状态同步机制
3. **技能库设计**: 模块化、可复用的技能定义

### 2.4 AIRSHIP

**论文地址:** [CUHK AIR](https://airs.cuhk.edu.cn/)  
**发布机构:** 香港中文大学  
**核心特点:** 开源具身 AI 框架

#### 架构特点

- 集成大语言模型与模块化机器人组件
- 高层推理与底层控制统一
- 支持导航和操作任务

#### 可借鉴点

1. **模块化设计**: 将机器人功能拆分为独立模块
2. **LLM 集成模式**: 大模型与机器人控制的接口设计

---

### 2.5 ROSA (NASA JPL)

**项目地址:** NASA JPL 开源项目  
**核心特点:** 自然语言查询和命令 ROS 系统

#### 功能

- 使用自然语言查询 ROS 系统状态
- 通过自然语言命令控制机器人
- 基于 LangChain 构建

#### 可借鉴点

1. **自然语言 ROS 接口**: 降低机器人编程门槛
2. **LangChain 集成**: 利用成熟的 Agent 框架

---

### 2.6 nanobot

**项目地址:** https://github.com/HKUDS/nanobot  
**核心特点:** 轻量级 Agent 框架

#### 说明

PhyAgentOS 基于 nanobot 开发，提供轻量级 Agent 基础框架。

#### 可借鉴点

1. **轻量级设计**: 适合资源受限的边缘部署
2. **简洁 API**: 易于集成和扩展

### 2.7 OpenClaw Robotics

**项目地址:** https://openclaws.io/blog/openclaw-robotics-embodied-ai  
**核心特点:** OpenClaw 在机器人领域的应用

#### 应用场景

- Unitree G1 人形机器人

- 空间记忆与物理世界交互

#### 可借鉴点

1. **OpenClaw 原生集成**: 与当前技术栈一致
2. **空间记忆**: Agent 具备环境空间认知能力

## 四、对 robot_openclaw 的借鉴建议

### 4.1 架构设计 (高优先级)

#### 建议 1: 采用分层架构
- **参考**: RoboOS 脑 - 小脑分层、PhyAgentOS 双轨系统
- **实施**: 
  - 高层：OpenClaw Agent (规划、决策)
  - 中层：安全验证层 (动作验证、包络检查)
  - 底层：ROS 2 执行层 (UR5 控制)

#### 建议 2: 实现 State-as-a-File 协议
- **参考**: PhyAgentOS
- **实施**: 定义 robot_openclaw 的状态文件格式
  - `robot_state.md`: 机器人当前状态
  - `action_queue.md`: 待执行动作
  - `safety_log.md`: 安全验证日志

### 4.2 安全机制 (高优先级)

#### 建议 3: 独立安全验证模块
- **参考**: PhyAgentOS Critic、ROSClaw 安全包络
- **实施**: 
  - 关节角度限制检查
  - 速度/加速度限制检查
  - 碰撞预测验证

#### 建议 4: 硬件 Watchdog
- **参考**: PhyAgentOS hal_watchdog.py
- **实施**: 独立的紧急停止监控进程

### 4.3 可扩展性 (中优先级)

#### 建议 5: 插件化驱动架构
- **参考**: PhyAgentOS hal/drivers/、RAI 模块化设计
- **实施**: 
  - 定义标准驱动接口
  - 支持动态加载新机器人驱动

#### 建议 6: 机器人能力自动发现
- **参考**: RAI rai_whoami
- **实施**: 从 URDF 自动提取机器人能力描述

### 4.4 开发体验 (中优先级)

#### 建议 7: 内置仿真支持
- **参考**: PhyAgentOS 仿真循环、RAI rai_sim
- **实施**: 强化 Gazebo 与 Agent 的集成

#### 建议 8: 基准测试套件
- **参考**: RAI rai_bench
- **实施**: 定义 robot_openclaw 的性能评估标准

### 4.5 高级功能 (低优先级)

#### 建议 9: 多机器人协同
- **参考**: RoboOS 实时共享内存、PhyAgentOS Fleet 模式
- **实施**: 预留多机通信接口

#### 建议 10: 经验学习机制
- **参考**: PhyAgentOS LESSONS.md
- **实施**: 失败案例库和成功技能库

## 五、总结

### 关键发现

1. **解耦是趋势**: 所有主流框架都采用认知层与执行层解耦的设计
2. **安全是底线**: 独立的安全验证机制是标配
3. **插件化是方向**: 支持快速扩展新硬件和新模型
4. **多机协同是未来**: 从单机器人向多机器人集群演进

### robot_openclaw 定位建议

基于调研结果，建议 robot_openclaw 定位为：

> **基于 OpenClaw 的轻量级机器人 Agent 框架，专注于 UR5 机械臂的智能化控制，采用分层架构和安全优先设计，支持插件化扩展和多机协同。**

### 下一步行动

1. 设计分层架构原型
2. 实现安全验证模块
3. 定义状态文件格式规范
4. 开发 UR5 驱动插件模板
